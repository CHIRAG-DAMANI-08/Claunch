<objective>
Implement automatic cross-session project memory synchronization and an interactive selector menu (similar to ccmanager) to select and open worktrees. Selecting a single worktree immediately opens it in a new tab without shifting focus from the menu tab (using `; focus-tab -t 0`).
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Implement directory junction syncing logic</name>
  <files>src/claude/memorySync.ts</files>
  <action>
  Create src/claude/memorySync.ts containing:
  - `syncMemoryJunctions(repoRoot: string, worktrees: Worktree[]): void` which:
    1. Computes the target project directory for the main repository.
    2. Ensures the main project's `memory` directory exists (creates it if needed).
    3. For each other worktree:
       - Computes its project directory.
       - Locates its `memory` directory.
       - If it doesn't exist, creates a directory junction pointing to the main project's `memory` directory (`fs.symlinkSync(src, dest, 'junction')`).
       - If it exists but is a normal directory, copies any new files/changes (like `MEMORY.md`) to the main project folder, deletes the target folder, and creates the junction.
       - If it's already a junction, skips it.
  </action>
  <verify>Junctions are created and linked correctly on Windows</verify>
  <done>src/claude/memorySync.ts is implemented and exports syncMemoryJunctions</done>
</task>

<task type="auto">
  <name>Task 2: Implement TUI Interactive Selector Menu</name>
  <files>src/terminal/interactiveMenu.ts</files>
  <action>
  Create src/terminal/interactiveMenu.ts:
  - Implements `startInteractiveMenu(worktrees: Worktree[], sessionStore: SessionStore, openTabsFn: Function): Promise<void>`
  - Uses Node's `readline` keypress listener to listen to raw stdin.
  - Lists discovered worktrees, highlighting the current one in green.
  - Displays checkbox list representing selections.
  - Key bindings:
    - Arrow keys or `j`/`k`: Navigate.
    - `Space`: Toggle multi-selection.
    - `Enter`: Launch all selected worktrees in a new Windows Terminal window (each in its own tab), then exit.
    - `o` or `L`: Launch the highlighted worktree immediately in a new tab in the current window, but append `; focus-tab -t 0` so the menu tab remains focused.
    - `q` or `Escape` or `Ctrl+C`: Exit.
  </action>
  <verify>TUI menu renders, responds to keypresses, and opens tabs as expected</verify>
  <done>src/terminal/interactiveMenu.ts is implemented and exports startInteractiveMenu</done>
</task>
