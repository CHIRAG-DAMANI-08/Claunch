<objective>
Implement automatic cross-session project memory synchronization.
By symlinking/junctioning the `memory/` subdirectory of each worktree's Claude project folder to the main repository's Claude `memory/` folder, all active worktree sessions will share a single, real-time updated `MEMORY.md` file.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Implement directory junction syncing logic</name>
  <files>src/claude/memorySync.ts</files>
  <action>
  Create src/claude/memorySync.ts containing:
  - `syncMemoryJunctions(repoRoot: string, worktrees: Worktree[]): void` which:
    1. Computes the target project directory for the main repository.
    2. Ensures the main project's `memory` directory exists (creates it if needed).
    3. For each other worktree:
       - Computes its project directory.
       - Locates its `memory` directory.
       - If it doesn't exist, creates a directory junction pointing to the main project's `memory` directory (`fs.symlinkSync(src, dest, 'junction')`).
       - If it exists but is a normal directory, copies any new files/changes (like `MEMORY.md`) to the main project folder, deletes the target folder, and creates the junction.
       - If it's already a junction, skips it.
  </action>
  <verify>Junctions are created and linked correctly on Windows</verify>
  <done>src/claude/memorySync.ts is implemented and exports syncMemoryJunctions</done>
</task>
