<task type="auto">
  <name>Task 2: Implement Windows Terminal Launcher</name>
  <files>src/terminal/openWindowsTerminal.ts</files>
  <action>
  Implement src/terminal/openWindowsTerminal.ts.
  Expose `openWindowsTerminal(specs: TabSpec[]): void`.
  It should construct arguments for `wt.exe`.
  Ensure it opens in a new window via `-w -1`.
  For each TabSpec:
    - Set the starting directory via `-d "${spec.path}"`.
    - Set the title via `--title "${spec.branch}"`.
    - Run the command. To avoid WT instantly closing on shell exit or execution error, we can run:
      `powershell.exe -NoExit -Command "${spec.command}"`
      or run it through powershell directly.
    - Chain tabs using semicolons. Semicolons must be escaped in PowerShell when we spawn the process!
    Since `child_process.spawn` without shell runs `wt` directly, we can pass individual arguments in an array.
    However, the Windows Terminal command-line parses semicolons inside its command parser.
    The syntax for wt.exe is:
    `wt -w -1 -d path1 --title title1 cmd1 ; new-tab -d path2 --title title2 cmd2`
    To do this via Node's `spawn`, we pass the arguments as an array:
    `['-w', '-1', '-d', 'path1', '--title', 'title1', 'cmd1', ';', 'new-tab', '-d', 'path2', '--title', 'title2', 'cmd2']`
    Let's test this behavior. We also need to check `wt` is spawned properly.
  </action>
  <verify>npx tsc --noEmit compiles cleanly</verify>
  <done>openWindowsTerminal compiles</done>
</task>
